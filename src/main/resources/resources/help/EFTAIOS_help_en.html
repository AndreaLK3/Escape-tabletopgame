Escape From The Aliens In Outer Space
User's manual

This manual only deals with the digital version of the game, for general rules
and / or information about the tabletop version, visit:
http://www.escapefromthealiensinouterspace.com/

Client operations, aka Play the Actual game

Preparations
  1. Set the port number in the main menu, the default 1337 works most of the times
  2. Click Start Client, then enter the ip / hostname of the server you want to join
  3. Wait for the game to start
  4. A popup will notify you when your turn starts
The main interface
The connection status, game status and turn number are displayed on the top row.
The left side of the screen contains the "tactical panel", showing the info you possess
about yourself and the other players, those data are automatically gathered by the game
for your convenience.

Under the global data about all the players are located the specific info about your character:
name, status, team. You may change your name by editing it and pressing enter.

Under your personal data, the turn status is displayed (who is currently playing)

The in-game chat interface is located at the bottom of the left panel, you can use it as any other
chat system you've seen before.

The right of the screen is occupied by the game map, it will show your position, as well as any
noteworthy event occurring during the game; to move your character, place noises, or turn on
the lights, you must select a cell by clicking it.

Finally, the bottom row of the interface contains the view object cards button, it shows a dialog
with the cards that are currently in your possession. When you have to play a card, simply bring
it up, click on the card, and press ok.
When you have drawn a new card, an hint is displayed in the bottom left of the screen.

The turn sequence
The turn sequence is unchanged from the tabletop version (advanced ruleses), but keep
in mind that turns have a precise time limit (around 60 seconds), and, if you take too much
time, you may lose the rest of your turn

Disconnection
The game abandon is permanent, meaning that you can not disconnect and the reconnect again
and continue the game; disconnected characters won't be removed from the map, but will not
figure among the winners.

Server operations

Preparations
  1. For the server, it's important to set the right networking mode
    • Choose Socket, RMI or Socket + RMI from the Server Settings row, in the main menu
    • Please note that the network mode chosen here will influence how the port will be used
  2. If you want to use a custom port (1337 is fine most of the times, though) for incoming connections, input it in the "port field", then hit the "set port" button
    • NOTE: if you are using RMI, the registry port (1099) will be listening, too.
    • OTHER NOTE: if you are using RMI + Socket, RMI will take over the displayed port (i.e. 1337), then Socket will try to listen on the next avaible one (i.e. 1338)
  3. Click "Start Server", and the server will be up and running in a few seconds.
  4. Now you may want to communicate your ip/hostname to your users, and let the game begin!
  • To allow others to connect, you may want to open the chosen ports on your firewall, the exact procedure vary between different operating systems.
        NOTE: we don't recommend opening the registry port (1099), as it could compromise the security of your system.
  • You may also want to allow connections from the whole internet (and not just your LAN). To do so, you must configure your router/gateway to forward said ports; again, the exact port-forwarding procedure vary between different machines. If you have trouble with port forwarding, these online tools can be helpful: http://www.yougetsignal.com/
        NOTE: we don't recommend forwarding the registry port, either.

The main interface
The server interface is rough, and is intended to simply display the server's logs, be it from the terminal
or inside a window.
The window does also display the server uptime, and offers buttons to:
Stop the server
Dump the logs (the file will show up in your working directory, usually the same as the executable .jar)

The server process will always try to go down gracefully (whether you stop it via gui or by sending a SIGINT),
by stopping all the games and then disconnecting all the users.

The server will automatically join new users to the same game; if said game is full / already running, a new game instance will be created, new users will be routed here, and so on.
The actual match will also start a fixed amount of time (default: 60 seconds, but it's customizable) after the minimum number of players is met (2 players).

Command line parameters

The application is intended to work in headless environments as well, but you need to use command line parameters to correctly set it up:
--gm-timeout [milliseconds]
 Set the duration after which the game will automatically start, given a minimum number of players waiting in the game room
--turn-duration [milliseconds] Set the absolute duration of a player's turn. When this time runs out, the computer will automatically concude the turn, in place of the user. The default is 60 seconds.
Note: using this option is not encouraged outside testing/debugging sessions.
--port [port number]
 Set the port to connect to / to listen on, just like the gui menu does
--server-address [ip / hostname]
 (Client only): set the default address to connect to (you don't have to choose it here, the cli will ask it anyway when you start the client)
--cli-client
 quickly start in command line client mode, bypassing the graphical menu
--cli-server
 quickly start in command line server mode, bypassing the graphical menu
--net-rmi
 use RMI instead of the default socket, on both the textual client and server
--net-combo
 (Server-only): use RMI and Socket simultaneously
--map-rotation [comma-separated map names]
 Set which game maps to play, instead of the default (Galilei). The avaible maps may vary based on your version of the program. Subsequent game instances will be played on subsequent maps on the list, the list is roll-over.
(red parameters are mandatory and mutually exclusive)

Q: My computer is an 80s IBM-PC, can I play too?
A: The protocol used by the Socket subsystem employs human-readable strings (kinda like SMTP does), this means that you can connect to an eftaios server via telnet / netcat and play, the interface is not so different from the cli-client provided by the application. Obviouly, the server must be hosted on a java capable computer.
